# v0.9 rejection remediation

v0.9 addresses both the clean-checkout rejection and the later production feature list. The Solidity protocol remains v0.6.0; application, indexer and SDK code are v0.9.0.

## Confirmed failures in the previous handoff

### Split dependency ownership

The root and browser used separate npm install paths and lockfiles. A prebuilt ZIP could run while a clean Vercel/GitHub checkout installed only one dependency tree.

**Resolution:** one root npm workspace, one lockfile and one Node/npm policy. `verify:clean-install` starts from an empty dependency state and proves install, doctor, artifact match, UI tests and production build.

### Unsupported Node range

The previous package accepted Node versions below the minimum required by its Vite major.

**Resolution:** `.nvmrc` pins `22.16.0`; root and workspaces require Node `>=22.12 <23` and npm `>=10 <11`; CI, Vercel and Docker use the same locked graph.

### False local readiness

The local supervisor could announce startup before proving the API and UI. A nested npm command also lost Vite host/port arguments.

**Resolution:** readiness is emitted only after EVM startup, source-matched artifacts, exact indexer trust anchors, API v0.9, seeded market, SSE and StockPair HTML all respond.

### Production localhost dependency

A static Vercel deployment could depend on an indexer at `127.0.0.1:8787`, which cannot exist in a user browser.

**Resolution:** `VITE_INDEXER_URL` may be blank. The browser then uses bounded direct RPC reads for network state, configured factory markets, portfolio balances, recent contract creation, ERC-20 probes, scanner evidence and Radar. It does not request static-origin `/api` routes or open EventSource. The initial deployment scan is capped at 100 blocks and later polls cover only newly produced blocks.

### Stale or substituted indexer acceptance

The published SDKs accepted arbitrary JSON services.

**Resolution:** browser and both SDKs require exact `X-StockPair-API-Version: 0.9.0`, expected JSON content type and bounded response bodies. SSE is opened only after a compatible REST handshake.

### Provider credential disclosure

Public network/health responses could include a complete authenticated server RPC URL.

**Resolution:** only a redacted transport descriptor is returned. Production RPC, explorer, WebSocket and Scout transports require TLS. Regressions prove path, query and hostname secrets are absent.

### Incomplete external trade action

External pools could be discovered without a complete safe quote/review/approval/execution path.

**Resolution:** V2/V3 adapters now require exact chain-matched nonzero runtime hashes for factory, router, WETH, pair/pool and V3 quoter. Quotes and route evidence are refreshed before signature. Exact allowance, confirmations, risk score, buy cap, slippage, price impact, recipient and explicit wallet review are enforced. Adapters remain disabled until real reviewed values are supplied.

### Missing durable alerts and reputation evidence

Launch alerts were evaluated but not delivered, and deployer/flow evidence was incomplete.

**Resolution:** the hosted indexer now supports bounded HMAC-signed webhooks, Discord/Telegram delivery, protocol alerts, deployer evidence scoring and self/reciprocal/circular/matched-size warnings. These are evidence-backed heuristics, not fraud findings.

## Acceptance gates

The release is rejected unless all of the following pass:

1. dependency-empty root `npm ci`;
2. workspace doctor and source/artifact/browser-ABI match;
3. full quick suite;
4. all 13 isolated deployed-bytecode scenarios;
5. clean-install verification;
6. production Vercel build with the indexer intentionally blank;
7. local EVM/indexer/SSE/UI readiness smoke;
8. runtime dependency audit;
9. hosted-template and OpenAPI/JSON parsing;
10. release manifest, secret hygiene, archive integrity and checksums.

## Not claimed

No hosted provider was deployed, no Robinhood mainnet transaction was sent, no external DEX adapter was production-enabled, no first mainnet market was created, and no user email service is included. Historical incident recovery and unknown vulnerabilities remain external boundaries.
